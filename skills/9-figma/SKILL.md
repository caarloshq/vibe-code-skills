# Figma — Execution

Creates frames in Figma based on the plan generated by `/8-plan-design`.
Lean by design: reads only what's needed, creates in the right order.

## Skill workflow position

```
  1-new-project     Technical foundation (architecture, CLAUDE.md)
  2-design-base     Visual foundation (globals.css, shadcn, styleguide)
  3-plan            Implementation planning (how to build it)

  5-new-component   Add/install components
  6-new-page        Build pages from designs
  7-doc-sync        Keep documentation in sync

→ YOU ARE HERE
```

## Step 1 — Read current page styles

Use Figma MCP to read **only the current page**. Don't scan the entire file.

Extract only:
- **Colors** — available color styles (name + hex value)
- **Typography** — available text styles (name, font, size, weight)
- **Components** — components already imported on the page (name + node ID)

If no styles are found on the page, pause and ask:
> "No styles found on the current page. Should I use the project's default tokens or do you prefer to import styles before continuing?"

Don't continue without styles or explicit user confirmation.

---

## Step 2 — Create the frames

Create frames in the order defined by the `/8-plan-design` plan.

Creation rules:
- **One frame per screen/state** — don't mix states in the same frame
- **Name by what the user sees**, not the technical state — e.g., `Buyback / No notes available`, not `Buyback / Empty state`
- **Use the styles found** — never invent colors or fonts outside what was read
- **Existing components have priority** — if it exists on the page, use it; don't recreate
- **Group by flow** — frames from the same flow stay close together, with a section label

### Creation order
- Base screen / default state first
- Alternative states (loading, empty, error)
- Corner cases last

---

## UX Writing — practices to apply in frames

When creating text, labels, titles, and messages in frames:

- **Screen titles**: describe what the user is doing — "Confirm buyback", not "Buyback"
- **Buttons**: verb + direct object — "Confirm operation", "Cancel buyback", not "OK" or "Confirm"
- **Empty states**: explain why it's empty and what to do — "You have no notes available for buyback. Wait for an operation to mature."
- **Error messages**: say what happened and the next step — "Could not process the buyback. Try again or contact support."
- **Field labels**: direct nouns — "Buyback amount", not "Enter the amount"
- **Tooltips**: one sentence, no period — "Calculated based on available balance as of today"
- **Avoid**: "Oops!", "Hmm...", overly informal language, generic placeholder text like "Lorem ipsum" or "Type here"
